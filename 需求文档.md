## 一、 项目背景

我们需要为某电商平台开发一个“智能售后客服 Agent”。
用户会通过对话框咨询商品问题、查询订单状态或申请退款。由于后端业务系统（订单中心）经常会出现网络波动，且大模型在提取参数时可能会产生幻觉（Hallucination），我们需要这个 Agent 具备极强的**路由分发能力**和**异常容错能力**。

------

## 二、 基础依赖设定（模拟后端 API）

请在你的 FastAPI 项目中，**先自行 Mock（模拟）两个本地业务接口**。这两个接口将作为 Agent 需要调用的 Tool：

1. **订单查询接口**
   - **路由：** `GET /api/order/status`
   - **要求头：** 必须携带 `Authorization: Bearer <token>`
   - **参数：** `order_id` (字符串，格式要求必须以 `ORD-` 开头)
   - **行为：** 格式正确且鉴权通过返回订单状态（如：已发货）；格式错误返回 `400`；鉴权失败返回 `401`。
2. **退款申请接口**
   - **路由：** `POST /api/order/refund`
   - **要求头：** 必须携带 `Authorization: Bearer <token>`
   - **参数：** `order_id` (字符串), `reason` (字符串)
   - **行为：** 随机模拟成功 (`200`) 与 后端系统崩溃 (`500`) 两种状态。

## 三、 核心架构与功能需求

请使用 **Python + FastAPI + LangGraph + 任意 LLM (如 OpenAI/DeepSeek 兼容接口)** 实现以下要求：

### 1. 多 Agent 路由机制 (Router)

- 接收用户的自然语言输入。
- 使用 LangGraph 构建状态机（StateGraph），设置一个**路由节点 (Router Node)**。
- 将用户请求精准分发到以下三个分支之一：
  - **闲聊/问答分支：** 处理简单的打招呼或常识问题。
  - **订单查询分支：** 识别订单号并调用查询 Tool。
  - **退款售后分支：** 提取订单号和退款原因，调用退款 Tool。

### 2. 用户鉴权与上下文透传 (重点)

- 对外提供的 Agent 对话接口（如 `POST /chat`）需要在 Header 中接收用户的 JWT Token（可以用任意假 Token 模拟）。
- **核心要求：** 大模型决定调用业务 Tool 时，必须能够将该 Token 从顶层 Request 成功透传到 Tool 内部发起的 HTTP 拦截器或请求头中。

### 3. Tool Calling 的异常打断与恢复 (重点)

Agent 在调用后端 API 时，**绝对不允许因为代码抛出 Exception 而导致整个对话服务崩溃 (500 Internal Server Error)**。你需要处理以下真实场景：

- **幻觉参数拦截：** 如果大模型瞎编了一个订单号（例如 `12345`，没有以 `ORD-` 开头），调用 API 会报 400。Agent 需要捕获此错误，并自然地用自然语言回复用户：“您提供的订单号格式似乎不正确，正确的格式应为 ORD- 开头，请您核对后再试。”
- **服务熔断兜底：** 当调用“退款接口”遇到后端返回 `500` 时，Agent 需要回复用户：“抱歉，当前退款系统繁忙，请稍后再试”，并在后台打印详细的 Error Trace。

## 四、 验收与加分项

### 💡 验收标准 (我们如何看你的代码)：

1. **State 的定义：** 你的 LangGraph `State` (TypedDict 或 Pydantic BaseModel) 定义是否清晰？是否有效管理了会话历史、提取的实体和错误状态？
2. **控制流逻辑：** 你是否真的使用了 Graph 的边 (Edges) 和条件边 (Conditional Edges) 来控制逻辑流转，而不是写了一堆臃肿的 `if-else`。
3. **健壮性：** 尝试输入刁钻的 prompt（如：“帮我退款，订单号是不知道”），看系统是否会优雅地提示参数缺失，而不是直接抛出空指针或网络异常。

### ⭐ 加分项 (Optional)：

- 实现了流式输出 (SSE，Server-Sent Events)。
- 接入了可观测性工具（如 LangSmith 或自定义的中断日志拦截器），能够清晰打印出 LLM 思考过程和 Tool 调用的耗时。
- 使用 Pydantic 严格约束了 Tool 的 Schema 并做了本地预校验（在真正发起 HTTP 请求前就拦截幻觉）。
